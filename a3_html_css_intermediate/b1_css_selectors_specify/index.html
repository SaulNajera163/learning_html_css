Which Cloud is better for our own requirements?

When a company wants to move on to AWS, they must consider factors such as required Cloud application components, preferred resource management tools, and any legacy IT infrastructure requirements.

Cloud-Based Deployment (All on the Cloud)

  • Definition: All parts of the application run in the Cloud, with no on-premises infrastructure.
  • Example: Hosting virtual servers, databases, and networking exclusively in AWS without any on-premises hardware.

On-Premises Deployment (All in my data center but some tools from AWS)

  • Definition: You keep everything in your own data center (on premises) while using technologies such as virtualization and resource management.
  • Example: Running applications entirely inside a company’s local data center, possibly with virtual machines to manage resources more efficiently.

Hybrid Deployment (My data center and AWS together)

Definition: A mix of both on-premises infrastructure and Cloud-based resources.

  • Cloud resources connect to on-premises systems, allowing you to keep some apps or data locally while benefiting from Cloud services for others.
  • Useful for scenarios with legacy applications that are not easily migrated, or for compliance requirements that mandate certain data remain on premises.

Scalability

The ability to increase or decrease resources to accommodate workload demands.

  1. Key Idea: More capacity is added as demand grows, ensuring applications continue to perform well.
  2. Example: A company running an online store increases the number of EC2 instances during a major sales event like Black Friday.

Elasticity

The ability to automatically scale resources both up and down in response to real-time demand.

Key Idea:Resources expand when needed and shrink when demand decreases, preventing waste and optimizing cost.

Example:A news website experiences a traffic spike due to breaking news, automatically adding more servers, then reducing them as traffic slows.
